Detailed Docker Setup

Option A: gvenzl/oracle-xe (Recommended)

Complete Setup 
# 1. Pull the Docker image
docker pull gvenzl/oracle-xe

# 2. Run Oracle XE container with persistence
docker run -d \
  --name oracle-xe \
  --restart unless-stopped \
  -p 1521:1521 \
  -p 8080:8080 \
  -e ORACLE_PASSWORD=yourSecurePassword \
  -v oracle-xe-data:/opt/oracle/oradata \
  gvenzl/oracle-xe

# 3. Monitor startup progress
docker logs -f oracle-xe

# 4. Test connection when ready
docker exec -it oracle-xe sqlplus system/yourSecurePassword@XE

Option B: Official Oracle Database

Complete Setup 
# 1. Pull the official Oracle image
docker pull container-registry.oracle.com/database/free:latest

# 2. Create volume for data persistence
docker volume create oracle-data

# 3. Run Oracle container
docker run -d \
  --name oracle-db \
  --restart unless-stopped \
  -p 1521:1521 \
  -p 5500:5500 \
  -e ORACLE_PWD=yourSecurePassword \
  -e ORACLE_CHARACTERSET=AL32UTF8 \
  -v oracle-data:/opt/oracle/oradata \
  container-registry.oracle.com/database/free:latest

# 4. Monitor startup (this takes 15-25 minutes)
docker logs -f oracle-db

# 5. Connect when you see "DATABASE IS READY TO USE!"
docker exec -it oracle-db sqlplus system/yourSecurePassword@FREEPDB1

Option C: Oracle 11g XE (Legacy)

Complete Setup

Apple Silicon Optimization

‚ö†Ô∏è
Performance Note for M1/M2/M3 Macs:

Oracle containers run under x86 emulation, expect:

  • 20-40% slower performance than native
  • Higher memory usage (add 2-4GB extra)
  • Longer startup times (2x normal)

Enable Rosetta 2 in Docker Desktop for best performance.

Troubleshooting

If you encounter issues while running Oracle Database in Docker, check below for common problems and solutions.

Common Issues

‚ùå
Container Fails to Start
  • Check logs: docker logs <container_name>
  • Ensure sufficient CPU and memory are allocated in Docker settings (recommend 4+ cores, 8GB+ memory).
  • Verify port 1521 is not already in use by another service.
‚ùó
Cannot Connect to Database
  • Ensure the container is running: docker ps
  • Double-check your connection string and credentials.
  • Check firewall rules that might block port 1521.
  • Confirm ports 1521 and 5500 are mapped correctly.
‚ö†Ô∏è
ORA-12541: TNS:no listener
Solution: Ensure the database has fully started and both 1521 and 5500 ports are mapped correctly.
‚ùó
High Memory Usage on Apple Silicon (M1/M2)
  • Enable Rosetta 2 in Docker Desktop settings.
  • Try using the gvenzl/oracle-xe image for better performance.
  • Allocate at least 12GB memory in Docker resources for smoother performance.
‚ÑπÔ∏è
Tip: For persistent data, always mount a Docker volume with -v to avoid data loss after container removal.

FAQs

How to change the Oracle password?

Use the following SQL command after connecting:

Change Password 
ALTER USER system IDENTIFIED BY newPassword123;

How to stop and remove the Oracle container?

Run the following commands:

Stop & Remove Container 
docker stop <container_name>
docker rm <container_name>
docker volume rm <volume_name>  # Optional: remove data volume

Where are the Oracle data files stored?

Data files are stored in Docker volumes. To view volumes, run:

List Docker Volumes 
docker volume ls
